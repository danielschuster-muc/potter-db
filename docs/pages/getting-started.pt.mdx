# REST vs. GraphQL
Potter DB: API oferece duas maneiras principais de acessar nossos dados mágicos: **REST** e **GraphQL**. Escolha uma que melhor atende suas necessidades e estilo de desenvolvimento.
- **REST**: Se você preferir uma abordagem direta e baseada em recursos, REST é o caminho a seguir. 
- **GraphQL**: Para consultas de dados mais flexíveis e precisas, GraphQL permite que você solicite exatamente o que você precisa e nada mais. Construa suas consultas para buscar dados com controle refinado.

## Limitação de taxa
Para manter a qualidade e previnir usos maliciosos do Potter DB: API, há um limite baseado em ip de **900 solicitações por hora** (15 solicitações / minuto). Se você atingir esse limite, receberá um `429 To Many Requests`.
Por favor, note que esses limites de taxa estão sujeitos a mudanças, então é essencial verificar a informação de limite de taxa mais recente na documentação ou via respostas da API.
