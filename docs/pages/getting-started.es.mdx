# REST vs. GraphQL

La API Potter DB: ofrece dos formas principales de acceder a nuestros datos mágicos: **REST** y **GraphQL**. Elige la que mejor se adapte a tus necesidades y estilo de desarrollo.
- **REST**: Si prefiere un enfoque directo, basado en recursos, los endpoints REST son el camino a seguir. Realizarás peticiones HTTP a URL específicas para recuperar datos.
- **GraphQL**: Para consultas de datos más flexibles y precisas, GraphQL le permite solicitar exactamente lo que necesita y nada más. Construye tus consultas para obtener datos con un control preciso.

## Limites de tarifa

Para mantener la calidad y evitar usos malintencionados de la API Potter DB: existe un límite basado en IP de **900 solicitudes por hora** (15 solicitudes / minuto). Si superas este límite, recibirás un mensaje `429 To Many Requests`.
Tenga en cuenta que estos límites de tarifa están sujetos a cambios, por lo que es esencial comprobar la información más reciente sobre límites de tarifa en la documentación o a través de las respuestas de la API.
